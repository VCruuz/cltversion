Gerenciamento de configuração de software

Controle de versão

Controle de versão é tipo uma ferramentas de software, que ajuda os usuários a ter um controle maior
sob as alterações feitas ao decorrer de todo o código, contribuindo na parte de achar certas
partes do código, na parte do salvamento do projeto e na de gerenciamento de tudo isso

Quando você faz uma alteração em um arquivo, o sistema de controle de versão anota essa mudança.
Assim, você pode ver um histórico de todas as edições e voltar a versões anteriores se precisar.
Além de que Se algo der errado ou se você precisar voltar a uma versão anterior do arquivo, o sistema permite
reverter para uma versão anterior facilmente.

Controle de Mudança

Controle de mudança é a maneira de gerenciar as alterações em um projeto para que tudo aconteça de forma organizada,
garantindo que cada mudança seja planejada, aprovada e documentada, evitando surpresas e problemas.
É como ter um plano para garantir que todas as modificações sejam feitas sem confusão e no tempo certo,
envolvendo planejar as mudanças, revisar e aprovar o que será feito, e documentar tudo para referência futura,
também ajudando a comunicar a todos os envolvidos sobre as mudanças

Integração Contínua

Integração contínua é uma prática usada no desenvolvimento de software para garantir que o código esteja sempre funcionando
da melhor forma possível, mantendo o código organizado e atualizado sempre, pois Sempre que alguém faz uma alteração no código,
essa mudança é automaticamente testada e integrada ao projeto principal.
Se por acaso algo der errado, você recebe um aviso rápido, o que ajuda a corrigir problemas antes que eles se tornem grandes demais.
